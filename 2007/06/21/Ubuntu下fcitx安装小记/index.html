Ubuntu下fcitx安装小记

发表于 2007-06-21 | 分类于 Linux

忍受不了SCIM了，于是决定投入fcitx的怀抱。
从http://www.fcitx.org上下载最新的3.4.1版，最开始的时候下载的是预编译好的包，下载回来，解压开，直接运行里面的fcitx.install，安装。执行fcitx，提示说SCIM正在运行，kill掉所有和SCIM有关的进程，再执行fcitx，可爱的小企鹅如期出现在桌面上，马上建个文档测试输入法，刚用Ctrl+Space把输入法打开，才按下一个字母fcitx就自动退出了，根据终端下的提示，是找不到智能拼音的数据库文件。想了想，再去弄这个也是个麻烦事情，就执行下载包里面的fcitx.uninstall把这个预编译版卸载掉。
再去下载回源码包自己编译。
在解压开的源码包里有一个名为autogen.sh的文件，试着执行了一下，出现一些编译方面的提示，直接./configure，然后make，等了一小会，编译出错，说是找不到/etc/X11/xpm.h，看样子是没有安装xpm相关的开发包，在 新立得 里面搜索xpm后确定是缺少了libgd2-xpm-dev 包，标记以便安装，此时系统自动选中额外的libgd2-xpm (2.0.33-2ubuntu5.1)、libgd2-xpm-dev (2.0.33-2ubuntu5.1)、libjpeg62-dev (6b-11)、libxpm-dev (1:3.5.4.2-0ubuntu3)四个包，并要求卸载libgd2-noxpm。确认以后，系统自动从源里面下载所需要的软件包并安装完毕。再次make，成功。 再 sudo make install 把fcitx安装到系统里。
到这里，fcitx的安装部分就算结束了。下面是一些配置工作。
首先，修改/etc/X11/xinit/xinput.d/zh_CN，把原来里面的东西(都是关于SCIM的东西)全部注释掉，然后试着去修改成fcitx需要的样子。
由于我并不知道该怎么去配置这些参数(我比较菜！)，还是cheaper兄提供了一个不错的主意，在这里着重感谢一下(来点掌声，再热烈一点~~~~)！
sudo apt-get -d install fcitx ，从源里面取回一个旧版本的fcitx的deb包，选择用 “归档管理器” 打开，看到里面有两个名为 control.tar.gz 和 data.tar.gz 的压缩文件和一个名为 debian-binary的文件。把两个压缩包提取出来，分别解压开来。
从control.tar.gz解压开得到的是一些包的配置文件。从里面的control文件可以看到一些关于包的描述信息，里面的conffiles文件里面仅有一行 /etc/X11/xinit/xinput.d/fcitx ，很明显，如果安装这个包，肯定会和这个文件有关系。再看从 data.tar.gz 解压开的东西，两个目录，etc的，按目录层层深入，就是前面的/etc/X11/xinit/xinput.d/fcitx ; 还有个usr的目录，自然就是fcitx的主程序了。现在我关心的仅仅是那个配置文件，打开以后得到：

<!–Code highlighting produced by Actipro CodeHighlighter (freeware)
http://www.CodeHighlighter.com/

–>XIM=fcitx
XIM_PROGRAM=/usr/bin/fcitx
XIM_ARGS=“”
GTK_IM_MODULE=XIM
QT_IM_MODULE=XIM
DEPENDS=“fcitx“

看样子，和先前SCIM的差不多，但是因为我先前并不知道fcitx的该怎么去设置，所以，我还是需要它的。因此，现在我直接把/etc/X11/xinit/xinput.d/zh_CN 里的内容全部替换成上面的就可以了。由于，我刚才编译的fcitx的可执行文件fcitx的目录是/usr/local/bin，并不是上面文件里的/usr/bin，这里，可以选择修改上面的路径，也可以在上面路径里建一个名为fcitx的到我的文件的链接。在效果上完全是一样的。
